Discuss the scope of var, let, and const

The "Var" or variable is a common and a very essential component of JavaScript. It's like a magical element which works differently other than "let" and "const" variables.
The "var" variable is kind of function level itself. It behaves as it is a global variable and the interesting fact is it can be accessed throughout the entire function in which it is declared.


let's look at an example.
function example() {
if (true) {
var x = 10;
}
console.log(x);
}
here, the "var" can be accessed throughout the entire function.


On the other hand, "let" an "const" are both block level variables which can't be accessed as "var" but there's a little difference between them.


The "let" allows to change the values of the variables even it is declared before but the "const" is unchangeable.

Tell us the use cases of null and undefined

"null" is generally an empty value which is used to show intentionally as empty. It is a variable which act or is used as it shouldn't have a value.

for example:

let selectedOption = null;

Here, the "let" variable is used to intentionally show an empty value.


On the other hand, "undefined" acts differently. It pops up when an user trying to get an output but the value isn't assigned yet. If the user trying to get an output without declaring a certain value the "undefined" will appear.


for example:

let age; console.log(age);

What do you mean by REST API?

The REST API in JavaScript works as a bridge for different softwares or applications to contact with one another over a network system or it can be called "The Internet".

It works as a communicator. Suppose JOHN ordered a PS5 on the internet. The provider will not directly deliver the PS5 to JOHN. The provider will get the order from Internet and then he will command the delivery-man to deliver the PS5 to JOHN.

In the same way, when an user requests a data to a server, the API of that server receive the request first and then send it to the main or Host server. Then the Host server checks if the request is valid or not. If the request is valid then it gives the requested data to the API and the API then delivers the data to the user.
